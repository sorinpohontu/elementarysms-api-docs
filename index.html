Authenticating API requests

Base URL

All API URLs referenced in this documentation start with the following base part:

http://elementarysms.example.com/api

You need to replace http://elementarysms.example.com with your own URL of your ElementarySMS installation.

Data Formats

Unless otherwise specified, all API calls will expect and return results as JSON data format.

Sending data to the APIs

By default, all API calls expect a JSON-formatted payload.

Here is a sample cURL call for /message/add:

curl -H "Content-Type: application/json" -X POST \
-d '{
     "api-key" : "XXSSDDESSQQ",
     "number": "+40722001002",
     "message": "This is a message sent through ElementarySMS API.\nHave a nice day :)"
    }' \
http://elementarysms.example.com/api/message/add

Receiving data from the APIs

By default, all API calls will return a JSON-formatted payload.

Here is a sample response from /message/add call:

{
  "status": "success",
  "uuid": "ea4b0fee-72ab-4d1f-9087-e8683ba355cf"
}

Special note about handling new lines

If you need to use a new line in your message body, replace the new line with \n.

Available methods

Message

This method will handle a message.

/message/add post

post

Add a new message for processing. On success, the UUID of newly added message will be returned.

post /message/add

Add a new message for processing. On success, the UUID of newly added message will be returned.

Body

Type: application/json

Schema:

{
  "$schema": "http://json-schema.org/schema",
  "type": "object",
  "properties": {
    "api-key": { "type": "string" },
    "number": { "type": "string" },
    "message": { "type": "string" },
    "device": { "type": "string" },
    "schedule_at": { "type": "date" }
  },
  "required": [ "api-key", "number", "message" ]
}

Example:

{
  "api-key": "XXYYZZTTZZAA",
  "number": "+40722001002",
  "message": "Hello John Doe, welcome to our site !"
}
